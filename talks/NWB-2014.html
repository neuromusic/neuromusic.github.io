Experimental Events data type

“...consists of times of events, along with values that correspond to the times. This data type can be used to describe: stimuli, trials or sweeps, time intervals, behaviors, and other events or conditions that occur during the experiment.”

INCF requirements for storing electrophysiology data
(PDF, 2014-11-03)

Event Requirements

  • Starting time of event monitoring
  • Ending time of event monitoring
  • Description of the events
  • Sequence of event times
  • Property values needed to describe each event.

However, behavior is not flat.

  • Stimulus events often have relationships behavior & neural activity depend on (e.g. Match to Sample tasks)
  • Behavioral events may be related through hierarchical structures (e.g. birdsong)

My point here is not to argue for a standard for representing behavior data, only that for even a relatively simple behavioral design, one needs to keep track of the relationships between events.
